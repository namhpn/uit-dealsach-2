# T0000 — Capture Repo Baseline State

## Goal

Verify the current repo structure, available commands, dependency state, and build/test status without changing application behavior.

## Allowed Areas

- `docs/Repo_Current_State.md`
- `docs/Known_Issues_And_Followups.md`
- `frontend/`
- `backend/`

## Do Not Touch

- Database migrations
- Runtime feature code
- Dependency files unless explicitly required and reported

## Requirements

- Inspect folder structure.
- Record available scripts.
- Run safe build/test commands where possible.
- Update `docs/Repo_Current_State.md`.
- Record failures as known issues instead of fixing them.

## Non-Goals

- Do not fix build failures.
- Do not install new libraries.
- Do not implement app features.

## Acceptance Criteria

- Repo state document reflects actual branch, folder structure, scripts, dependency status, and build/test results.
- Any failures are recorded with exact commands and output summary.
