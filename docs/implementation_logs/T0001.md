## Requirements

### Docker Compose command cleanup

- Update Docker Compose so documented commands no longer require:

```bash
  --env-file backend/.env
```

* After this ticket, normal commands should use:

  ```bash
  docker compose up -d --build
  docker compose down
  docker compose run --rm app sh -lc 'cd backend && php spark list'
  docker compose run --rm app sh -lc 'cd backend && php vendor/bin/phpunit'
  ```

* Do not remove CodeIgniter’s ability to use `backend/.env` for runtime app configuration.

* Do not commit real secrets.

* If Compose variable substitution currently depends on `backend/.env`, replace that dependency with safe defaults or another non-secret documented approach.

### Dockerized frontend npm commands

* Add a Docker Compose service for frontend Node commands.

* The service must run npm commands inside Docker.

* The service must use the local `frontend/` folder so `frontend/node_modules/` is created locally and reused between runs.

* Do not use a named or anonymous Docker volume for `frontend/node_modules`.

* Do not commit `frontend/node_modules/`.

* Support these commands:

  ```bash
  docker compose run --rm frontend npm install
  docker compose run --rm frontend npm run build
  docker compose run --rm frontend npm run dev -- --host 0.0.0.0
  ```

* Expose the Vite dev server port if needed.

### Frontend dependency baseline

* Install or verify frontend dependencies through the Dockerized frontend service.
* If `npm install` creates `frontend/package-lock.json`, keep it only if npm is the intended package manager.
* If the repo clearly intends pnpm instead of npm, stop and report the package-manager mismatch instead of switching package managers silently.
* `docker compose run --rm frontend npm run build` must pass.

### Backend PHPUnit baseline

* Fix the non-zero PHPUnit result from T0000.
* Prefer making the default backend test command pass without requiring code coverage.
* Do not install or require Xdebug/PCOV unless the existing project explicitly requires coverage for normal local tests.
* After the fix, this command must exit 0:

  ```bash
  docker compose run --rm app sh -lc 'cd backend && php vendor/bin/phpunit'
  ```

### AGENTS.md and template path cleanup

* Update `AGENTS.md` so references to templates use the actual paths:

  ```text
  docs/templates/Codex_Handoff_Template.md
  docs/templates/Completion_Report_Template.md
  docs/templates/Ticket_Template.md
  ```

* Remove outdated references to template files at the root of `docs/`.

* Update all Docker command examples in `AGENTS.md` to remove `--env-file backend/.env`.

### Known issues cleanup

* Close KI-0001 only if Dockerized frontend dependency install/build works.
* Close KI-0002 only if backend PHPUnit exits 0.

## Acceptance Criteria

* `docker compose config` succeeds without `--env-file backend/.env`.
* `docker compose up -d --build` succeeds without `--env-file backend/.env`.
* `docker compose down` succeeds.
* `docker compose run --rm app sh -lc 'cd backend && php spark list'` succeeds.
* `docker compose run --rm app sh -lc 'cd backend && php vendor/bin/phpunit'` exits 0.
* `docker compose run --rm frontend npm install` succeeds or reports a clear package-manager mismatch.
* `docker compose run --rm frontend npm run build` succeeds.
* `frontend/node_modules/` exists locally after install but is not committed.
* No named or anonymous Docker volume is used for `frontend/node_modules`.
* `AGENTS.md` references template files under `docs/templates/`.
* `AGENTS.md` Docker commands no longer require `--env-file backend/.env`.
* `docs/Repo_Current_State.md` is updated.
* `docs/Known_Issues_And_Followups.md` is updated.
* No runtime feature code is changed.
