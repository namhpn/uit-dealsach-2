# T0003 — Public Catalog Read APIs and Offer Eligibility

## Goal

## Requirements

### 1. Public API routes

Add public JSON API routes for catalog read behavior.

Required endpoints:

```text
GET /api/public/books
GET /api/public/books/{bookId}
GET /api/public/discovery
GET /api/public/filters
```

The implementation may use different controller class names, but route behavior must be documented in the completion report.

### 2. Centralized public catalog read service

Implement reusable backend read logic for:

* Active public books.
* Latest observation per offer.
* Freshness calculation using a 48-hour threshold.
* Current eligible-offer calculation using EO-001 through EO-009.
* Lowest observed available price from currently eligible offers.
* Offer count for book cards.
* Public no-price status indicator priority:

  1. `Chưa có liên kết mua hợp lệ`
  2. `Tạm hết hàng`
  3. `Giá tham khảo cũ`
  4. `Chưa có ưu đãi`
* Vietnamese reference-price disclaimer:

```text
Giá tham khảo được ghi nhận gần đây, vui lòng kiểm tra lại tại nơi bán trước khi mua.
```

The same eligibility logic must be reused by discovery, search/list, filters, and book detail responses.

### 3. Offer grouping for book detail

For `GET /api/public/books/{bookId}`, return tracked public offers grouped into:

* `purchasable`
* `unavailable`
* `stale_reference`
* `missing_valid_seller_link`

Rules:

* Purchasable offers must satisfy all eligible-offer conditions.
* Unavailable offers must not include an active Buy action.
* Unavailable offers must display `last_available_price` when one exists.
* Stale Active offers must appear in `stale_reference` and must not include an active Buy action.
* Active offers with fresh available price but missing or invalid affiliate destination must appear in `missing_valid_seller_link` and must not include an active Buy action.
* Pending Review, Inactive, and Removed / Invalid offers must be hidden from public catalog responses.
* Public responses must not expose exact observation timestamps.

### 4. Book-level price history

For `GET /api/public/books/{bookId}`, return book-level price history using observation-time eligibility.

Rules:

* Use stored observation-time facts from `price_observations`.
* Do not use current offer state to remove historical chart records.
* Group by Vietnam date.
* Return the lowest observation-time-eligible price per day.
* If no observation-time-eligible price exists for a day, either omit the day or mark it unavailable consistently.
* Do not expose exact observation timestamps in public responses.

### 5. Search, filter, sort, and pagination

For `GET /api/public/books`, support:

Query parameters:

```text
q
category
author
publisher
retailer
availability
min_price
max_price
sort
page
per_page
```

Required behavior:

* Empty search returns default discovery-style listing.
* Default public page size is 12.
* Archived books are excluded.
* Search supports title, author, publisher, category, and ISBN.
* Search is case-insensitive.
* Search supports Vietnamese accented and unaccented input.
* Partial matching is required.
* ISBN search ignores hyphens and spaces.
* Typo tolerance is out of scope.
* Token-order-insensitive matching is out of scope.
* Search ranking order:

  1. exact ISBN match
  2. exact title match
  3. title prefix match
  4. title partial match
  5. author match
  6. publisher match
  7. category match
  8. fallback order
* Fallback order is book title ascending, then creation order ascending.

Availability filter values:

```text
all
available_now
stale_reference
temporarily_unavailable
missing_valid_seller_link
no_tracked_offer
```

Do not add a broad `no_available_offer` filter.

Price range filtering:

* Uses the book’s lowest observed available price from currently eligible offers.
* Excludes books with no currently eligible offer.
* Does not use stale or unavailable prices.

Invalid query parameters must return clear Vietnamese validation errors.

### 6. Discovery endpoint

For `GET /api/public/discovery`, return structured sections for:

* `featured_books`
* `recent_price_drops`
* `popular_clicked_deals`

Required behavior:

* Featured books use Admin-curated featured flag from the existing book data.
* Recent price drops use book-level lowest eligible price.
* Recent price drops use the last 7 days.
* Recent price drops rank by largest absolute VND drop.
* A book with no previous eligible price is not counted as a price drop.
* If multiple drops occur within 7 days, use the largest single observation-cycle drop.
* Ties sort by most recent drop first, then book title ascending.
* If the previous cycle is unavailable, compare against the most recent earlier cycle with an eligible price.
* A drop caused only by the previous lowest offer becoming ineligible must not count unless an eligible observed price actually decreased.
* Because click-event persistence is not implemented in this ticket, `popular_clicked_deals` may return an empty section with a Vietnamese empty-state message and must be documented as pending future click-tracking work.

### 7. Filters endpoint

For `GET /api/public/filters`, return public filter metadata:

* Active categories.
* Authors from active public books.
* Publishers from active public books.
* Active retailer platforms that have public-relevant offers.
* Availability buckets.
* Supported sort values.

Archived categories must not appear as selectable public filters.

### 8. Response shape and formatting

Public API responses must:

* Use stable JSON keys suitable for frontend consumption.
* Return prices as whole-number VND values.
* Include Vietnamese user-facing labels/messages where labels/messages are returned.
* Include the required price disclaimer near price/offer data.
* Avoid implying real-time price guarantees.
* Avoid cart, checkout, payment, order, shipping, voucher, coupon, review, rating, or direct-purchase semantics.
* Avoid exposing admin-only exact last checked timestamps.

### 9. Tests

Add backend tests covering:

* Public routes return JSON and expected HTTP statuses.
* Search excludes archived books.
* Search supports accented and unaccented Vietnamese input.
* ISBN search ignores hyphens and spaces.
* Pagination defaults to 12.
* Availability filters return the correct bucket behavior.
* Price range filters use only currently eligible offers.
* Book-card status priority is applied correctly.
* Book detail returns purchasable, unavailable, stale reference, and missing valid seller link sections where seeded data supports them.
* Stale and unavailable offers do not expose an active Buy action.
* Missing/invalid destination offers do not expose an active Buy action.
* Lowest eligible price uses EO-001 through EO-009.
* Price history uses observation-time eligibility and does not disappear because of current stale/unavailable states.
* Public responses do not expose exact observation timestamps.
* Recent price drops follow FR-016 through FR-021 where seeded data supports verification.
* Popular clicked deals returns a safe empty section until click-event persistence exists.
