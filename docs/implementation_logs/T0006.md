## Requirements

- Read `./AGENTS.md` before implementation.
- Remove the duplicate standalone homepage search bar. The header search is enough.
- Keep a clear homepage search entry point through the banner CTA:
  - It may focus the header search input, or
  - It may navigate to `/search`.
- Header category chips must be loaded from `GET /api/public/filters`, not from a hardcoded runtime `navCategories` list.
- Header category chip labels and slugs must match API filter categories.
- Header category chip active state must be derived from the current URL/location, not sticky local state.
- Returning to `/` must leave no category chip visually pressed.
- Clicking a header category chip must navigate to `/search?category={slug}`.
- Restore the original homepage section order:
  1. Banner carousel
  2. Featured category shelves
  3. Recent price drops
  4. Popular clicked deals
  5. DealSach guidance
- Implement featured category shelves using API-backed book data only.
  - Prefer grouping `GET /api/public/discovery` `featured_books.items` by `category_slug`.
  - Use `GET /api/public/filters` category order when available.
  - Do not restore original mock `featuredCategories` as runtime data.
- Demo seed data must include enough `books.is_featured = 1` records across multiple seeded categories to make category shelves meaningful.
- Demo seed categories and frontend category chips must not include slugs absent from `GET /api/public/filters`.
- Demo seed data must include enough successful `affiliate_redirects` across multiple books and retailers to visibly exercise popular clicked deal ranking, redirect count, and top retailer metadata.
- Popular clicked deal cards must still use API-backed `popular_clicked_deal.redirect_count_7d` and `top_retailer`.
- Recent price-drop cards must still use API-backed VND drop metadata.
- Fix KI-0008 so fresh long-running Docker app containers serve HTTP traffic without manual `chown`.
- Docker writable ownership normalization must:
  - Run automatically during container startup or equivalent Docker lifecycle.
  - Be idempotent.
  - Preserve `php-fpm` as the app container’s long-running process.
  - Avoid broadly changing ownership of the whole repository mount.
  - Limit ownership normalization to needed writable runtime paths, primarily `/var/www/html/backend/writable`.
- Preserve one-off backend commands:
  - `docker compose run --rm app sh -lc 'cd backend && php spark migrate'`
  - `docker compose run --rm app sh -lc 'cd backend && php vendor/bin/phpunit'`
- Preserve existing Dockerized frontend commands.
- Preserve T0004/T0005 behavior:
  - Public catalog data is API-backed.
  - Search remains API-backed through `GET /api/public/books`.
  - Filters remain API-backed through `GET /api/public/filters`.
  - Book detail remains API-backed through `GET /api/public/books/{bookId}`.
  - Buy buttons still use `/go/offers/{offerId}`.
  - Non-purchasable offers do not show active Buy actions.
- Public UI must not imply DealSach sells books directly.
- Public UI must not mention carts, checkout, payment, shipping, vouchers, coupons, reviews, or ratings.
- Required Vietnamese price disclaimer must remain near price-bearing areas:

```text
Giá tham khảo được ghi nhận gần đây, vui lòng kiểm tra lại tại nơi bán trước khi mua.
```
