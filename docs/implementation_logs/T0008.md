# T0008 — Authenticated Wishlist APIs and Frontend Integration

## Goal

Allow registered users to log in through the existing email-code auth APIs from the frontend and manage a persistent wishlist from book cards, book detail, and a dedicated wishlist page.

## Dependencies

- T0007 — Backend Email Verification and Session Foundation must be reviewed and merged.
- `./AGENTS.md`
- `docs/requirement-doc.md`

  - `POST /api/auth/email-code/verify`
  - `GET /api/auth/me`
  - `POST /api/auth/logout`
- Existing public catalog APIs and frontend catalog pages.

## Requirements

- Read `./AGENTS.md` before implementation.
- Add a new backend migration for wishlist persistence.
- Add a `wishlist_items` table with at least:
  - `id`
  - `user_id`
  - `book_id`
  - timestamps
  - unique constraint on `(user_id, book_id)`
  - foreign keys or equivalent model-level referential safeguards to users and books.
- Add `WishlistItemModel`.
- Add `WishlistService` for all wishlist business logic.
- Add JSON wishlist endpoints:
  - `GET /api/user/wishlist`
  - `GET /api/user/wishlist/books/{bookId}`
  - `POST /api/user/wishlist/books/{bookId}`
  - `DELETE /api/user/wishlist/books/{bookId}`
- Wishlist endpoints must require a valid authenticated session.
- Guest requests to wishlist endpoints must return Vietnamese JSON with HTTP 401 and the existing response shape:
  - `status`
  - `message`
  - `data`
  - `errors`
- `GET /api/user/wishlist` must return the authenticated user’s wishlist entries.
- Wishlist list entries must include enough book-card data for frontend display:
  - book id
  - title
  - author
  - publisher
  - category
  - category slug
  - cover image
  - current public price/status summary when the book is still public
  - wishlist added timestamp
  - archived marker when the book is archived.
- `GET /api/user/wishlist/books/{bookId}` must return whether the authenticated user has wishlisted that book.
- `POST /api/user/wishlist/books/{bookId}` must:
  - add an Active public book to the authenticated user’s wishlist.
  - treat an already-wishlisted book as a no-op success.
  - reject archived or nonexistent books with Vietnamese JSON.
- `DELETE /api/user/wishlist/books/{bookId}` must:
  - remove the wishlist item when present.
  - treat missing items as a no-op success.
- A user may wishlist a book only once.
- Existing wishlist entries for books later archived must remain visible in the wishlist with an archived label.
- Archived books must not be newly added to a wishlist.
- Do not change public catalog responses unless adding internal helper methods with no public API behavior change.
- Add backend database and feature tests for:
  - schema constraints.
  - authenticated add/list/status/remove flow.
  - duplicate add no-op behavior.
  - guest rejection.
  - archived book add rejection.
  - archived existing wishlist item remains listed and marked archived.
  - deactivated/invalid session rejection if the T0007 auth service exposes that state.
  - public catalog and Buy-flow smoke stability after the new migration.
- Extend frontend API helpers for:
  - auth request-code, verify-code, `/me`, logout.
  - wishlist list, status, add, remove.
  - `credentials: "include"` for authenticated requests.
- Add frontend auth state management under `frontend/src/app/auth/`.
- Add a Vietnamese email-code login/register UI that:
  - can be opened from the header account button.
  - can be opened when a guest clicks a wishlist action.
  - requests an email verification code.
  - verifies the code.
  - refreshes current user state after successful verification.
  - supports logout.
  - does not display or assume the verification code in the public UI.
- Update header behavior:
  - account button opens login/register UI when guest.
  - account area shows authenticated email and logout affordance when logged in.
  - wishlist button opens login/register UI for guests and navigates to `/wishlist` for authenticated users.
- Add `/wishlist` frontend route and `WishlistPage`.
- `WishlistPage` must:
  - prompt guests to log in.
  - load and display authenticated user wishlist entries.
  - show Vietnamese loading, error, and empty states.
  - allow removing books from wishlist.
  - link active books to their detail pages.
  - visually mark archived wishlist books.
- Update book-card wishlist affordances on homepage/search/discovery cards:
  - guest click prompts login.
  - authenticated click adds/removes wishlist item.
  - card navigation must still work when clicking outside the wishlist button.
- Update book detail wishlist affordance:
  - guest click prompts login.
  - authenticated user can add/remove the current book.
  - archived or unavailable-for-wishlist cases show a Vietnamese explanation.
- User-facing messages and validation errors must be Vietnamese-first.
- Use `Asia/Ho_Chi_Minh` for wishlist timestamps where application code formats or compares time.
- Do not implement price alerts in this ticket.
- Do not implement Admin management in this ticket.
- Do not add external auth, state-management, email, queue, or UI dependencies.
- Do not create Playwright, browser-driven, screenshot, or UI capture tests.
