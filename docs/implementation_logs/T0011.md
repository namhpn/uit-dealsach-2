## Requirements

* Add deterministic backend alert evaluation, runnable from a Spark command such as:

  * `php spark alerts:evaluate`
* Evaluate only alerts satisfying evaluable-alert rules:

  * Alert status is `Active`.
  * Owning user is active.
  * Account-level alert emails are enabled.
  * Related book is Active.
  * Alert is not expired.
  * Required target/baseline state exists for the alert type.
  * At least one current eligible offer exists when price comparison is needed.
  * Covers EA-001 through EA-008.
* Reuse centralized current eligible-offer behavior for EO-001 through EO-009.
* Implement target-price alert notification behavior:

  * Covers AL-004 through AL-011.
  * No immediate email at creation.
  * No repeated email for unchanged below-target conditions.
  * Update comparison, last-notified price, and notification count only when rules permit.
* Implement new-lowest-price alert notification behavior:

  * Covers AL-012 through AL-018.
  * Pending baseline is established without email.
  * Tied lowest offers in one evaluation pass create one notification event/email, not duplicates.
* Implement account-level alert email suppression:

  * Covers AL-035 through AL-043.
  * Suppressed conditions must not create Triggered events.
  * Suppressed conditions must not write alert emails.
  * Suppressed conditions must not increment notification count.
  * Suppressed conditions must not update last notified price.
* Write mock alert emails to existing `outbound_emails`:

  * Vietnamese subject/body.
  * Distinct `email_type` values for target-price and new-lowest alert emails.
  * Include book title, observed price, target price or baseline context, reference-price disclaimer, and affiliate disclosure.
  * Link to DealSach book detail or offer section, not directly to an external seller.
  * Include a separate disable-alert link.
  * Covers AL-049 through AL-065 within mock email scope.
* Add email deal-link tracking:

  * Persist generated email deal links and click events.
  * Deal-link clicks must be recorded separately from Affiliate Redirects.
  * Deal-link route must land on DealSach book detail or offer section, not an external affiliate destination.
* Add disable-alert link handling:

  * Token protected.
  * Expires after 30 days.
  * Single-use.
  * Affects only one alert.
  * Expired, invalid, or already-used links return Vietnamese messages.
  * If the alert is already Disabled, return a Vietnamese no-op confirmation.
  * Usage is logged as an alert event.
  * Covers AV-022 through AV-029 and AL-063 through AL-064.
* Implement failed mock email behavior:

  * Failed attempt is recorded.
  * One immediate retry is attempted in the same evaluation run.
  * Failed sends do not update last notified price.
  * Failed sends do not increment notification count.
  * Covers AL-044 through AL-048 and AL-066 through AL-072 within mock email scope.
* Enforce the 3-successful-notification limit:

  * Only successful mock email submissions count.
  * After the third successful notification, alert transitions to `Auto-paused`.
  * Record an alert event.
* Add or refine a registered account/settings page:

  * Route such as `/account`.
  * Uses existing auth state and auth dialog.
  * Shows the authenticated user’s email and role/status.
  * Provides logout access.
  * Provides account-level alert email preference control.
  * Explains in Vietnamese that disabling alert emails does not change individual alert statuses.
* Keep `/alerts` preference controls working after any shared API/helper changes.
* Add focused backend tests for alert evaluation, email records, deal-link click tracking, disable links, suppression, failed retries, and auto-pause behavior.
* Add frontend build-safe account/settings integration without browser/UI automation tests.
* Do not add new Composer or npm dependencies.

## Acceptance Criteria

* [ ] `php spark alerts:evaluate` runs and prints a concise Vietnamese or developer-readable summary of evaluated, triggered, emailed, suppressed, failed, baseline-set, expired, and auto-paused alerts.
* [ ] Target-price alerts trigger according to AL-004 through AL-011 and avoid duplicate unchanged-condition emails.
* [ ] New-lowest alerts trigger according to AL-012 through AL-018 and set pending baselines without immediate email.
* [ ] Account-level suppression prevents Triggered events, mock emails, notification-count changes, and last-notified-price updates.
* [ ] Mock alert emails are written to `outbound_emails` with alert/book/price/link metadata.
* [ ] Email deal-link clicks are persisted separately from Affiliate Redirects.
