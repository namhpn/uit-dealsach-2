## Requirements

* Add Admin-only backend dashboard/report APIs:

  * Guests receive Vietnamese 401 JSON.
  * Non-admin authenticated users receive Vietnamese 403 JSON.
  * Admin users may read dashboard reports.
  * Read-only report views must not create Admin audit records.
* Dashboard report window:

  * Default to the last 7 days.
  * Use `Asia/Ho_Chi_Minh`.
  * Do not implement custom dashboard date ranges.
  * Covers DR-007 and DR-008.
* Report grouping rules:

  * Group metrics by stable entity identifier.
  * Display current entity names.
  * Mark archived entities as archived in dashboard display.
  * Historical events remain counted after related books, retailers, merchants, offers, or users are archived, renamed, invalidated, disabled, or deactivated.
  * Covers DR-001 through DR-004.
* Dashboard summary cards:

  * Total Affiliate Redirect count.
  * Total Email Deal Link Click count.
  * Total Redirect Failure count.
  * Active Alert Status count.
  * Evaluable Alert count.
  * Email-suppressed Active Alert count.
  * Auto-paused Alert count.
  * Expired Alert count.
  * Alert Email Sent count.
  * Alert Email Failed count.
  * Admin mutation audit count.
* Affiliate redirect reports:

  * Count Affiliate Redirects in the default 7-day window.
  * Group Affiliate Redirects by book.
  * Group Affiliate Redirects by retailer platform.
  * Include current book/retailer names and archived markers.
  * Covers DR-009 through DR-011.
* Email engagement reports:

  * Count Email Deal Link Clicks separately from Affiliate Redirects.
  * Group email clicks by book and alert type where available.
  * Do not treat email deal-link clicks as affiliate redirects.
  * Covers DR-012.
* Redirect failure reports:

  * Count Redirect Failures separately from Affiliate Redirects.
  * Group by reason/category when available and by book/offer when available.
  * Covers DR-013.
* Alert reports:

  * Active Alert Status Count.
  * Evaluable Alerts Count using the same evaluable-alert rules from T0011.
  * Email-Suppressed Active Alerts.
  * Auto-paused Alerts.
  * Expired Alerts.
  * Email Sent Count.
  * Email Failed Count.
  * Covers DR-014 through DR-020.
* Price-change report:

  * Compute book-level Price Change Summary.
  * Definition: latest book-level lowest eligible observed price minus previous book-level lowest eligible observed price.
  * Use current eligible-offer rules for latest/previous comparable prices.
  * Mark books with no comparable previous value as unavailable/not enough data.
  * Offer-level price change dashboard summary remains out of scope.
  * Covers DR-021 through DR-023.
* Admin audit report:

  * Include recent Admin mutation counts and recent audit entries.
  * Use existing Admin audit persistence.
  * Do not duplicate the full `/admin/audit` page; dashboard should summarize and link to it.
* Frontend Admin dashboard UI:

  * Add route such as `/admin/dashboard` or make `/admin` show the dashboard summary.
  * Add Admin navigation entry for Reports/Dashboard.
  * Show Vietnamese labels and guidance.
  * Display summary cards.
  * Display tables for grouped reports.
  * Display simple dependency-free charts, such as CSS bar charts or proportional rows, without adding chart libraries.
  * Handle loading, empty, error, 401, and 403 states.
  * Keep Admin pages usable at 768px minimum width.
* Backend tests:

  * Cover Admin authorization.
  * Cover default 7-day window behavior.
  * Cover Affiliate Redirect grouping by book and retailer.
  * Cover email deal-link clicks being separate from Affiliate Redirects.
  * Cover redirect failures separately.
  * Cover alert status/evaluable/suppressed/auto-paused/expired/email sent/email failed metrics.
  * Cover book-level price-change summary.
  * Cover archived entity markers and historical event counting.
  * Cover that read-only dashboard APIs do not create audit records.
* Frontend build must pass.
* Do not add new Composer or npm dependencies.
