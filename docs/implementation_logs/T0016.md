## Requirements

* Fix the local browser CORS failure for Vite frontend requests from `http://localhost:5173` to backend/API at `http://localhost`.
* Configure CodeIgniter CORS without using wildcard origins with credentials.
* Allow credentialed local API requests because auth/Admin flows use the HTTP-only `dealsach_session` cookie.
* CORS must cover at least:

  * `Origin: http://localhost:5173`
  * methods `GET`, `POST`, `PATCH`, `DELETE`, `OPTIONS`
  * headers `Accept`, `Content-Type`, `X-Requested-With`, and `Authorization`
  * `Access-Control-Allow-Credentials: true`
* Apply CORS consistently to `api/*` routes before and after request handling, including preflight `OPTIONS`.
* Add or update backend feature coverage proving auth preflight and credentialed auth endpoints return the required CORS headers.
* Verify admin email-code login from the Vite frontend can request a code without the browser CORS preflight error.
* Resolve KI-0011 only if it is still recurring:

  * First try to reproduce the phpMyAdmin `8080` port conflict using the normal documented Compose command.
  * If the conflict recurs, make the phpMyAdmin host port configurable with a safe default, for example `${PHPMYADMIN_HOST_PORT:-8080}:80`.
  * Document how to run with an alternate port such as `PHPMYADMIN_HOST_PORT=8081 docker compose up -d --build`.
  * If the conflict does not recur after clean verification, do not change Compose solely for KI-0011; instead update the known issue with evidence that it was a one-time local environment conflict.
* Resolve KI-0012 by defining and implementing concrete category display metadata:

  * Add a migration for `categories.display_label` as nullable string.
  * Add `categories.display_description` as nullable text.
  * Add `categories.display_order` as integer with default `0`.
  * Update Category model allowed fields.
  * Update Admin category list/detail/create/update APIs to read, validate, persist, and audit these fields.
  * Update Admin category UI to view and edit these fields.
  * Update public category/filter responses to expose metadata without breaking existing `id`, `name`, and `slug` fields.
  * Public UI may use `display_label` where helpful, but must fall back to `name`.
  * Seed deterministic metadata for existing demo categories.
* Resolve KI-0013 by adding deterministic dashboard demo seed scenarios:

  * Seed representative active, suppressed, auto-paused, and expired price alerts.
  * Seed alert email sent and failed outcomes.
  * Seed email deal links and email deal-link clicks.
  * Seed recent Admin audit summaries.
  * Keep the data deterministic and inside the fixed last-7-days Vietnam-time dashboard window where appropriate.
  * Do not alter dashboard report semantics unless tests expose an existing defect.
* Keep all product UI/messages Vietnamese-first.
* Preserve existing JSON envelope conventions:

  * `status`
  * `message`
  * `data`
  * `errors`
* Preserve existing Admin authorization behavior:

  * guests receive 401
  * non-admin registered users receive 403
  * active Admin sessions can access Admin APIs
* Close or update only the known issues actually resolved or reclassified by this ticket.
* Do not add new runtime dependencies.

## Manual Verification

1. Start a clean disposable stack:

   ```bash
   docker compose -p dealsach_t0016 up -d --build
   ```

2. Expected result: the stack starts. If phpMyAdmin port `8080` is unavailable and KI-0011 recurs, verify the documented alternate-port workflow after the Compose fix.

3. Run migration and seed:

   ```bash
   docker compose -p dealsach_t0016 run --rm app sh -lc 'cd backend && php spark migrate && php spark db:seed DealSachDemoSeeder'
   ```

4. Expected result: migrations complete, seed completes, category metadata exists, seeded Admin exists, and dashboard demo rows are inserted.

5. Verify CORS preflight:

   ```bash
   curl -i -X OPTIONS http://localhost/api/auth/email-code/request \
     -H 'Origin: http://localhost:5173' \
     -H 'Access-Control-Request-Method: POST' \
     -H 'Access-Control-Request-Headers: content-type'
   ```

6. Expected result: response includes `Access-Control-Allow-Origin: http://localhost:5173`, `Access-Control-Allow-Credentials: true`, allowed methods, and allowed headers.

7. Verify actual auth request includes CORS headers:

   ```bash
   curl -i -c /tmp/dealsach-t0016-auth-cookies.txt \
     -H 'Origin: http://localhost:5173' \
     -H 'Content-Type: application/json' \
     -d '{"email":"admin@dealsach.test"}' \
     http://localhost/api/auth/email-code/request
   ```

8. Expected result: request succeeds with Vietnamese JSON envelope and CORS headers.

9. Inspect mock outbox for the seeded Admin code:

