## Requirements

### Search autocomplete

* Add autocomplete suggestions to the public search input.
* Suggestions must appear while the user types a non-empty query.
* Suggestions must be loaded asynchronously from the backend/public catalog layer.
* Suggestions must include enough information for users to recognize a book:

  * title
  * author
  * primary category
  * lowest eligible price when available
  * status label when no eligible price exists
* Selecting a suggestion must navigate to the book detail page.
* Pressing Enter in the search input must continue to perform the normal search navigation.
* The existing search results page must continue to support search, filtering, sorting, and pagination.
* The autocomplete response must exclude archived books.
* The autocomplete response must respect the same Vietnamese accented/unaccented and partial-matching behavior as public search.
* Limit suggestions to a small bounded result set, for example 6 items.
* Show Vietnamese empty/loading/error states where appropriate.
* Avoid new runtime dependencies.

### Office 365 SMTP email delivery

* Configure backend email sending to use Office 365 SMTP values from environment variables:

  * `SMTP_ADDRESS`
  * `SMTP_PORT`
  * `SMTP_USERNAME`
  * `SMTP_PASSWORD`

* Do not commit real SMTP credentials.

* Preserve `outbound_emails` records for audit/demo visibility.

* Verification-code emails must be sent through SMTP when SMTP configuration is complete.

* Alert emails must be sent through SMTP when SMTP configuration is complete.

* If SMTP configuration is incomplete, keep a safe local/mock behavior so existing local tests and demo workflows do not crash.

* Email sending failures must be recorded in existing outbound-email/alert notification behavior.

* Do not expose OTP codes in API responses.

* Update the seeded Admin account email to:

  ```text
  24521102@gm.uit.edu.vn
  ```

* Admin login must continue to use the existing email-code flow.

* Existing email verification limits, code hashing, session creation, and neutral login/register responses must remain unchanged.

* Existing alert email semantics must remain unchanged:

  * email deal links land on DealSach first
  * users click Buy separately
  * disable-alert links remain separate
  * email link clicks are tracked separately from affiliate redirects
