# T0019 — Commerce-First Neubrutalist Homepage Refresh

## Goal

Implement the clarified DealSach homepage refresh as a commerce-first, API-backed, controlled Neubrutalist discovery page using the existing public catalog systems, with small discovery metadata/API enhancements and no homepage admin system.

* T0018 completed and reviewed.
* Existing public discovery API:

  * `GET /api/public/discovery`
  * `GET /api/public/filters`
  * `GET /api/public/books`
  * `GET /api/public/books/suggestions`
* Existing frontend homepage:

  * `frontend/src/app/pages/HomePage.tsx`
  * `frontend/src/app/shared.tsx`
  * `frontend/src/app/api.ts`
* Existing public catalog backend:

  * `backend/app/Libraries/PublicCatalogService.php`
  * `backend/app/Controllers/PublicCatalogController.php`
  * `backend/tests/feature/PublicCatalogApiTest.php`
* Existing seeded demo data:

  * `backend/app/Database/Seeds/DealSachDemoSeeder.php`
* Current design/decision context:

  * Homepage is shopping/discovery-first, brand second.
  * Use controlled Neubrutalism.
  * Keep three current API discovery sections.
  * Keep hero banners frontend-configured.
  * Do not add homepage admin CRUD.
  * Card CTA text may say `Đến nơi bán →`, but it means navigating to `/book/{id}`, not external seller redirect.

## Requirements

### Homepage product direction

* The homepage must prioritize shopping/discovery over brand-only presentation.
* The visual style must remain controlled Neubrutalist:

  * sharp corners,
  * black borders,
  * hard shadows,
  * flat colors,
  * Be Vietnam Pro,
  * no gradients,
  * no soft shadows,
  * no rounded cards/buttons/inputs.
* Keep the existing homepage section order unless the existing code structure requires a minor ordering adjustment:

  1. Hero banner carousel.
  2. Featured books grouped by category.
  3. Recent price drops.
  4. Popular clicked deals.
  5. How it works.

### Color-token cleanup

* Refactor frontend color tokens so red is no longer named `secondary`.
* Add a true `secondary` token with `#496173`.
* Add a red commercial/error token such as `priceAlert` or `dealRed` using `#ba1a1a`.
* Update existing frontend usages intentionally:

  * price text and price-drop badges use `priceAlert`/`dealRed`;
  * actual secondary styling uses `secondary`;
  * error states may use `priceAlert`/`error`.
* Preserve existing visual intent where possible.

### Hero banner behavior

* Keep hero banners frontend-configured.
* Do not add a backend/admin banner system.
* Keep the CTA behavior “as is” in principle: CTA should navigate into the existing public flow rather than require a new backend feature.
* Add enough config structure so each banner can have explicit behavior without new APIs:

  * link to `/search`,
  * link to a section anchor,
  * or navigate to an existing route.
* Do not add direct external seller redirect from hero banners.

### Discovery API metadata

* Extend discovery section payloads with small metadata only:

  * `subtitle`
  * `cta_label`
  * `cta_href`
  * optional `window` metadata for time-windowed sections.
* Keep the existing three discovery sections:

  * `featured_books`
  * `recent_price_drops`
  * `popular_clicked_deals`
* Do not replace the discovery response with a generic dynamic `sections[]` architecture in this ticket.
* Do not add new search sorts such as `price_drop_7d` or `popular_7d`.

### Category shelves

* Keep featured books grouped by category.
* Use category `display_label` with fallback to `name`.
* Render category `display_description` where available.
* Continue using current `is_featured` and category ordering.
* Do not add per-category featured ordering.
* Do not add homepage-specific category admin controls.

### Product cards

* Homepage product cards must link to `/book/{id}`.
* Product card CTA text should be `Đến nơi bán →`, but it must navigate to `/book/{id}`.
* Do not implement direct seller redirect from homepage cards.
* Do not add `best_buy_action` to cards.
* Wishlist buttons must remain on homepage cards and continue opening auth dialog for unauthenticated users.
* Do not batch wishlist status in this ticket.
* Do not add price-alert creation controls to homepage cards.

### Product card content

* Compact cards should show:

  * cover/fallback,
  * category,
  * title,
  * author,
  * current eligible price or public status label,
  * offer count,
  * wishlist action.
* Larger deal cards should additionally show publisher if space allows.
* Product cards should not show individual price disclaimers.
* Use a section-level or module-level disclaimer instead.
* If there is no eligible current price, show the public status label.

### Original/reference price behavior

* Add API support for “original price” display where possible using the highest current eligible price across all reseller offers for the same book.
* Do not invent MSRP or fake original price.
* Add a field with clear semantics, for example:

  * `highest_eligible_price`
  * or `reference_highest_eligible_price`
* Display strikethrough/reference price only when:

  * `lowest_eligible_price` is not null,
  * the highest eligible price is not null,
  * and highest eligible price is greater than lowest eligible price.
* For recent price-drop cards, continue showing VND savings from `price_drop.amount`.
* Do not add discount percentage.

### Seed/test data

* Ensure seeded demo data contains enough eligible multi-retailer price variation to display highest-vs-lowest reference price on at least some homepage cards.
* Preserve existing deterministic test expectations where possible.
* Update tests if public card DTO shape changes.

### Empty states and fallbacks

* Keep existing empty-state behavior for:

  * no featured books,
  * no recent price drops,
  * no popular clicked deals.
* Add clear section subtitles/empty messages that explain:

  * recent price drops are based on eligible price observations in the recent window;
  * popular deals are based on successful affiliate redirects in the recent window.
* Do not hide a section silently just because it is empty.

### Accessibility and UX

* Preserve keyboard navigation and focus visibility.
* Preserve autocomplete header behavior from T0018.
* Do not regress search submission behavior.
* Carousel arrows must remain accessible with `aria-label`.
* Book cover fallback must remain functional.

## Non-Goals

* Do not add `best_buy_action`.
* Do not add new search sorts.
