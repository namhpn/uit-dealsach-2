## Requirements

### 1. Preserve production ProductDetailPage foundation

- Keep ProductDetailPage API-backed.
- Continue loading detail data through existing `fetchBookDetail(id)`.
- Continue loading related-book fallback through existing discovery data if currently used.
- Preserve existing wishlist behavior.
- Preserve existing price-alert behavior.
- Preserve existing Buy-flow link behavior.
- Do not reintroduce mock product data from `frontend-original/`.
- Do not add browser, Playwright, Puppeteer, screenshot, or visual-capture testing.

### 2. Product title and metadata

- Keep the current ProductDetailPage layout.
- Restore stronger `frontend-original/` title typography:
  - tighter letter spacing,
  - stronger wall-of-text feel,
  - still responsive and readable.
- Do not enlarge the title so much that it breaks long Vietnamese titles.
- Metadata below title must remain compact and secondary.
- Breadcrumb must remain visually quiet.

### 3. Best price panel

- Keep the current emerald-tinted best-price block.
- Keep the current hard border and hard shadow.
- Show:
  - large current best price,
  - strikethrough reference/original price when available from existing API data,
  - red deal badge when reference price is greater than current best price.
- Use existing `summary.highest_eligible_price` as the reference/original display source.
- Do not invent MSRP.
- Do not add new backend fields.
- Do not add discount percentage unless already implemented locally and derived only from existing `highest_eligible_price` and `lowest_eligible_price`.
- Preferred red badge text:
  - `TIẾT KIỆM {amount}` when `highest_eligible_price > lowest_eligible_price`.
- Change the current best-price text color to black.
- Seller row price color must match the best-price current price color.

### 4. Wishlist and price-tracking action alignment

- Change unsaved wishlist label from `Lưu sách` to `Thêm vào Wishlist`.
- Saved state may remain `Đã lưu`.
- Align wishlist and price-tracking buttons visually with the best-price panel.
- The action stack must not look detached or vertically awkward beside the price panel.
- Recommended best practice:
  - make both action buttons equal width,
  - keep consistent height,
  - align to the top of the price panel on desktop,
  - stack full-width under price panel on mobile if horizontal space is tight.
- Keep the price-tracking hero CTA as an anchor to the price alert section.
- Preserve auth-dialog behavior for unauthenticated wishlist actions.

### 5. Seller / market price rows

- Keep seller section grouping.
- In seller rows, remove redundant status chips such as:
  - `Giá tham khảo cũ`,
  - `Có thể đến nơi bán`,
  - other chips that duplicate the enclosing section meaning.
- Keep the `Rẻ nhất` badge for the cheapest purchasable seller.
- Put merchant/reseller name and reseller product title on the same visual line where space allows.
  - Example structure: `Sách Hay 24h / Cha giàu cha nghèo`
  - On narrow screens, allow wrapping without truncating critical seller name.
- Keep retailer/platform name visually primary.
- Seller price color must match the best-price current price color.
- Keep hard-bordered unavailable/stale/missing-link row treatment.
- Keep disabled action blocks for non-purchasable rows.

### 6. Seller section grouping

- Keep the existing separate groups:
  - purchasable market prices,
  - temporarily unavailable,
  - stale reference,
  - missing valid seller link.
- Hide groups that have zero rows.
- Do not show empty-state cards for hidden empty groups.
- If all groups are empty, keep the existing page-level empty state.

### 7. Price alert module

- Improve clarity between the target-price alert and the no-target “new lowest / price drop” alert.
- Current problem: the two actions feel too close and can be confused as variants of the same submit.
- Recommended best practice:
  - visually separate the target-price form from the general price-drop action,
  - place the target-price form in the primary alert block,
  - place the no-target alert in a secondary bordered callout below or beside it,
  - use distinct helper copy for each.
- Change `Theo dõi giá thấp mới` to `Thông báo khi giảm giá`.
- Keep target-price submit label as `THEO DÕI GIẢM GIÁ`.
- Keep target-price input behavior.
- Keep unauthenticated submit behavior opening the auth dialog.
- Keep `3px` input border and visible focus styling.

### 8. Technical details

- Change section title from `Thông tin kỹ thuật` to `Thông tin chi tiết`.
- Preserve existing metadata fields and `Chưa cập nhật` fallback behavior.
- Do not change backend metadata schema.

### 9. Related books

- Improve the related-books layout by studying the `frontend-original/` visual rhythm.
- Current issue to resolve: cards are evenly spaced but feel awkward and do not fit the section horizontally.
- Recommended best practice:
  - keep exactly 4 related books on desktop when available,
  - use a contained row/card shelf that visually spans the section,
  - avoid awkward empty horizontal gaps,
  - ensure each card has consistent width,
  - prefer a full-width 4-column grid or original-like flex row with controlled card width,
  - avoid changing shared card behavior globally unless scoped by props.
- Preserve click behavior to `/book/{id}`.
- Preserve wishlist behavior inside related cards.
- Do not add backend related-books logic.

### 10. Disclaimer block

- Keep the current visual disclaimer container.
- Replace current disclaimer copy with the fuller `frontend-original/` trust copy.
- Copy must explicitly mention:
  - prices are reference prices,
  - prices may change at seller site,
  - DealSach does not sell books directly,
  - DealSach does not handle payment,
  - DealSach does not handle delivery,
  - DealSach does not handle returns,
  - DealSach does not provide order support,
  - some links may be affiliate links.
- Keep copy in Vietnamese.
- Do not add affiliate disclosure beside every CTA.

### 11. Preserve shared behavior

Because `frontend/src/app/shared.tsx` is shared across Homepage, SearchPage, WishlistPage, and ProductDetailPage:

- Do not regress T0022 Homepage visual rhythm.
- Do not regress T0021 SearchPage card behavior.
- If related-book card layout needs adjustment, prefer ProductDetailPage-specific wrapper styling or scoped props.
- Do not globally alter compact card behavior unless all call sites remain visually correct.
- Do not change public API types unless required by TypeScript build.
